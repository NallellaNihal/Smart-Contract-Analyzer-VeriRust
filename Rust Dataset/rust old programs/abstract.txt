Smart contracts must exhibit high reliability, as even a minor programming error can result in significant security and financial repercussions. This work presents VeriRust, an automated framework for the analysis and verification of smart contract programs developed in Rust. The system accepts a Rust source file as input and autonomously executes assertion injection, formal verification, coverage analysis, and result creation. The approach seeks to minimize manual labor in contract verification.

The framework initiates when the user executes a script such as ./verify_rust.sh filename.rs. Once the input file is identified, the script creates a well-structured results directory, saves a backup copy of the original source code, and then uses an injection module to insert reachability assertions in conditional expressions and loop statements. Embedded statements exhibit the availability of both valid and invalid execution paths. The modified program is then submitted to the Kani model checker for formal verification. Outcomes are saved for review. The original source file is restored after verification, and any temporary files created during the execution are removed to ensure a clean workspace. The framework also performs branch and condition coverage analysis using the LLVM coverage tools, producing summary and report files for each contract.

The obtained reports provide the original code, the adapted source with assertions, the output from Kani, the verification summary, and the LLVM coverage metadata. These artifacts develop the transparency, reproducibility, and evaluative significanceof the verification process. VeriRust offers a systematic method to examine the functionality of Rust smart contracts, identify accessible code paths, and enhance assurance in the correctness of the contracts. The present execution relies on additional tools, including Kani, Cargo, and LLVM coverage support. The framework presents a feasible approach for the automatic formal verification of Rust smart contracts.
